Design · Section 1 — the tactical model

Consolidating your six answers. Two aggregates: Lesson grows a child Exercise collection + a Grade behavior; Attempt is a new persisted root recording each graded submission.

Lesson aggregate — 1 aggregate → 2 tables
Lesson (root)
Id · UnitId · Title · Position · IsPublished
+ EnsureCompletable() · + Grade(answers) → GradingResult
Exercise (entity)
Id · Position · Prompt · Choices[] · CorrectChoice 🔒
+ IsCorrect(choice) → bool

Grading lives on Lesson.Grade (owns exercises + keys + threshold). Extract to a GradingService only when a cross-lesson rule appears. Key sealed behind Exercise.IsCorrect.

Attempt aggregate — NEW, persisted (first write path)
Attempt (root)
Id · LearnerId · LessonId (by id) · SubmittedAt
Answers[] · Score · Outcome
Answer (owned)
ExerciseId · SelectedChoice · WasCorrect

Created already-graded from a GradingResult. Persisted every time (pass or fail) → the history Slice 3 reads. References Lesson by id only.

Value objects
Score — correct/total as %, owns MeetsThreshold(t)
Prompt · reuse Title · typed ids (ExerciseId, AttemptId)
GradingResult — { Score, Outcome, per-exercise correctness }
Outcome — enum { Passed, Failed }

One type now, polymorphism-ready: Exercise is modelled so a second type (translate, listen-type) is an additive subtype overriding IsCorrect — no change to Attempt, Grade, or the flow. Threshold is a single global domain constant (e.g. 80%) — one place to change; Lesson.Grade compares the Score against it via Score.MeetsThreshold.
