How deep should the content catalog go?

Learning is a supporting subdomain — the discipline here is restraint. Deep enough to be a real tree, shallow enough to avoid gold-plating. This choice also sets the aggregate boundaries we'll draw on the next screen.

A

Flat — Course → Lesson

Two entities. The leanest thing that still points at real content.

📘 Course · “Spanish”
📄 Lesson · “Basics 1”
📄 Lesson · “Basics 2”
📄 Lesson · “Basics 3”

+ Simplest possible.   − No grouping for Slice 3's skill-tree / “unlock next unit”.

B

Grouped — Course → Unit → Lesson recommended

A real 3-level tree. Ordering and “next lesson / next unit” become meaningful — but we skip Duolingo's Section and Skill layers.

📘 Course · “Spanish”
📂 Unit · “Basics”
📄 Lesson · “Greetings”
📄 Lesson · “The verb ser”
📂 Unit · “Food”
📄 Lesson · “At the café”

+ Real tree, sets up Slice 3 cleanly, still lean.   − One more entity than A.

C

Deep — Course → Section → Unit → Skill → Lesson

The real Duolingo shape. Five levels.

📘 Course
🗂️ Section
📂 Unit
🎯 Skill
📄 Lesson

+ Most realistic.   − Gold-plating a supporting domain — five aggregates to map, seed, and test for a skeleton.
