Keeping the live scheduler out of the existing E2E suite
All 4 hosts (League, LeagueSettlement, Streak, Engagement) inject a shared FakeTimeProvider and jump it (Jan→Mar). A live scheduler on that same clock would fire mid-test → settle weeks the test didn't ask for → races & double-moves. Fix: each factory adds one line —
builder.UseSetting("Leagues:Settlement:Enabled", "false");
The two league factories require it; Streak/Engagement get it too for uniformity (would be harmless no-ops otherwise).
⚖ One open choice — a live end-to-end smoke test?
We could add one E2E with the scheduler enabled (own DB/factory): earn in W1 → advance the clock one interval past the boundary → poll /me/league until it shows auto-settlement, with no manual POST. It proves the real wiring, but testing a background loop through the Host needs poll-with-timeout (mild flakiness).
My recommendation: skip it this slice. The mechanism test (deterministic) + the policy test already cover both risks; a flaky timing test buys little. Easy to add later. — but it's your call.
