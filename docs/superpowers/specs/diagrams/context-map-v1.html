Proposed Context Map (v1)

Boxes = bounded contexts. Arrows = relationships, labeled with DDD integration patterns. The gold box is our CORE.

The integration patterns I used (the real vocabulary)

Upstream / Downstream

  • Published Language + Open Host Service: Learning publishes a well-defined LessonCompleted event; any context can subscribe. Learning doesn't know who listens.
  • Conformist: everyone accepts Identity's UserId as-is.
  • Anti-Corruption Layer (ACL): Learning/Engagement translate Billing's entitlement model into their own terms, shielding the core.

Symmetric

  • Partnership: Engagement & Social co-evolve leaderboards (leagues need the social graph; social shows league standings).
  • Shared Kernel (avoided): tempting to share a "User" model everywhere — we deliberately DON'T, to keep contexts independent.
