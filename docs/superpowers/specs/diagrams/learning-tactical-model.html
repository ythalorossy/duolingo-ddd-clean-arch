Value objects (inherit BuildingBlocks.Domain.ValueObject)
  • CourseId, UnitId, LessonId — Guid wrappers. Kill primitive obsession, give value-equality, serve as keys. Exactly like LearnerId. (not up for debate — this is the pattern)
  • Title — one shared VO: non-empty, trimmed, bounded length. Or… a plain string, if we want the supporting domain leaner. decision
  • Position — kept a plain int (an ordering index). A VO would be gold-plating here.
