Phase 3 · What do we build first?

A "walking skeleton" = the thinnest end-to-end slice that exercises every layer of the architecture, built around the CORE.

Build-order options

A

Engagement walking skeleton (recommended)

Thinnest core slice: award + read XP, end-to-end. Stub the lesson trigger & fake the current user. Validates the WHOLE architecture and models the core first.

B

Learning engine first

Build real lessons/exercises/grading so there's genuine input, then Engagement. More upfront work before the core is validated.

C

Identity/foundation first (bottom-up)

Auth & accounts first. Safe-feeling, but you spend early effort on a GENERIC subdomain and validate the core last.

Proposed sequence after the skeleton

1. Engagement skeleton (XP) → 2. grow Engagement (streaks, then leagues) → 3. Learning (real lessons replace the stub) → 4. Identity (real auth replaces the fake user) → 5. Notifications / Social / Billing as needed. We always keep a running, demoable system.
