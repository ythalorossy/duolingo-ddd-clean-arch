Q5 — Where does the grading logic live?

Grading needs three things — the exercises, their correct answers, and the pass threshold — and produces a Score + Outcome. All three inputs live on Lesson; the result is recorded by Attempt. So who orchestrates the tally?

Shared by all three options: each Exercise entity answers IsCorrect(choice) → bool (tell-don't-ask). The raw correct answer 🔒 never leaves the Exercise — the grader only ever asks "is this choice right?". So encapsulation of the key is equal across A/B/C; the real question is orchestration + separation of concerns.

A — A GradingService domain service spec pre-named this

handler → GradingService.Grade(lesson, answers)
  ↳ loops exercises · calls IsCorrect · builds Score/Outcome
→ new Attempt(learner, lesson, answers, result)

The first explicit domain service in the codebase — the textbook home for logic that spans two aggregates (Lesson = truth, Attempt = record). Highest new-building-block learning value. Risk: a thin service can drift toward anemic if it's "just a loop."

B — Lesson.Grade(answers) → GradingResult DDD-purest

handler → lesson.Grade(answers) → GradingResult (VO)
→ new Attempt(learner, lessonId, answers, result)

Behavior lives on the aggregate that owns every input (exercises, keys, threshold). No anemic service; nothing to hand around. A lesson is a gradeable unit, so grading is arguably its core behavior. Risk: mixes "content" and "assessment" responsibilities on one root.

C — Attempt.Grade(lesson)

handler → new Attempt(learner, lessonId, answers)
→ attempt.Grade(lesson) → sets Score/Outcome

Tell-don't-ask on the Attempt. But the Attempt must reach across into the Lesson's exercises to grade itself — an aggregate reading deep into another aggregate's internals. Weakest boundary story of the three.

My lean: genuinely torn between A and B. On pure DDD merit I'd pick B (behavior with its data, no service smell). But your notes say the learning goal can be the tiebreaker — and A builds a domain service, a named building block this repo doesn't have yet (and the Slice-1 spec literally pre-named it). If you want that muscle, A is well-justified. C I'd set aside.
