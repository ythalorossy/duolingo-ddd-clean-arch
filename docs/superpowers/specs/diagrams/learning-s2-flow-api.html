Design · Section 2 — data flow & API surface

Completion stops being asserted and becomes earned. The bare Slice-1 POST /complete is replaced by an attempt-submission endpoint; a new read presents the questions.

The grading flow (POST an attempt)
POST /lessons/{id}/attempts { answers: [ {exerciseId, choice}, … ] } → load Lesson (+ exercises) unknown → 404 → lesson.EnsureCompletable() unpublished → 409 → validate answers cover the exercises missing/unknown → 400 → lesson.Grade(answers) → GradingResult (Score vs global threshold) → new Attempt(learner, lessonId, answers, result) → SaveChanges (persist attempt — pass OR fail) → if Passed: publish LessonCompleted → Engagement XP → 200 { attemptId, score, outcome, perExercise: [{exerciseId, wasCorrect}] }
Endpoints
POST /lessons/{id}/attempts — grade a submission (above). Replaces Slice-1 POST /lessons/{id}/complete. Pass and fail are both 200 (a graded result isn't an error). Repeatable: each pass awards XP again — dedup is Slice 3.
GET /lessons/{id} — present the lesson's exercises to answer: prompt + choices, no correct answer. New read model (the 2D map node-tap view). CQRS read port; the key never enters the DTO.
GET /courses — unchanged catalog read from Slice 1.
removed POST /lessons/{id}/complete + the CompleteLesson command — superseded by SubmitAttempt (which publishes LessonCompleted on a pass).
Persistence (learning schema grows)
learning.Exercises — child table of the Lesson aggregate (FK LessonId, CorrectChoice column). LessonRepository.GetByIdAsync now eager-loads exercises.
learning.Attempts (+ owned Answers) — new root table; new IAttemptRepository.Add + first SaveChanges.
Migration AddExercisesAndAttempts — new tables + extend the HasData seed with multiple-choice exercises on the published lessons (unpublished lesson stays exercise-light to keep the 409 path).
