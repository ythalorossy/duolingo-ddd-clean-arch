Nuance 1 · reference by identity CourseId / UnitId are plain id columns. We deliberately do not add a Unit.Course navigation property — that would let code hop across aggregate boundaries (a train-wreck traversal). A DB-level foreign key for integrity is optional; the object reference is always id-only.
Nuance 2 · aggregate ≠ table A root with owned children spans multiple tables but stays one aggregate — e.g. Engagement's XpAccount → xp_accounts + applied_awards. Here each root is a single entity with only value-object properties, so it's a clean 1 aggregate = 1 table. That's a property of this model, not a general law.
Module rule All three live in a new learning schema — one schema per module. No cross-schema JOIN to engagement; the only link across modules stays the LessonCompleted event.
