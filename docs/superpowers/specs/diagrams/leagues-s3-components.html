Design · §2 — Components

Three new types + one new repo query. New Host type = a driver (fine). New Application/Domain types reference nothing infrastructural (dependency rule holds — NetArchTest stays green).

1 · The BackgroundService — pure mechanism (Host)
protected override async Task ExecuteAsync(CancellationToken stoppingToken)
{
    using var timer = new PeriodicTimer(options.Value.PollInterval, clock);   // ②
    do
    {
        try
        {
            using var scope = scopeFactory.CreateScope();                // ①
            var mediator = scope.ServiceProvider.GetRequiredService<IMediator>();
            await mediator.SendAsync(new SettleDueLeagueWeeks(), stoppingToken);
        }
        catch (Exception ex) when (ex is not OperationCanceledException)  // ③
        {
            log.LogError(ex, "Settlement tick failed; retrying next interval.");
        }
    }
    while (await timer.WaitForNextTickAsync(stoppingToken));   // OCE on shutdown → loop ends
}
① A BackgroundService is a singleton → it can't hold the scoped IMediator/DbContext. Open an IServiceScopeFactory scope per tick. (The classic .NET trap.)
② PeriodicTimer takes the injected TimeProvider → FakeTimeProvider.Advance() drives it in tests; no real waiting. do/while = run once on startup (fast catch-up), then every interval.
③ Swallow + log a failed tick. An unhandled throw in ExecuteAsync silently kills the service for the Host's whole life — the loop must survive a transient DB blip.
2 · The policy command (Application)
public sealed record SettleDueLeagueWeeks : IRequest<Unit>;

public sealed class SettleDueLeagueWeeksHandler(
    ILeagueStandingRepository standings, IMediator mediator, TimeProvider clock)
    : IRequestHandler<SettleDueLeagueWeeks, Unit>
{
    public async Task<Unit> HandleAsync(SettleDueLeagueWeeks _, CancellationToken ct)
    {
        var current = LeagueWeek.Containing(clock.GetUtcNow());
        var due = await standings.GetDistinctEndedWeeksAsync(current, ct); // ascending
        foreach (var week in due)                  // oldest first → chain holds
            await mediator.SendAsync(new SettleLeagueWeek(week.Start), ct); // idempotent
        return Unit.Value;
    }
}
3 · The query (Domain interface + Infra impl)
// ILeagueStandingRepository (Domain)
Task<IReadOnlyList<LeagueWeek>> GetDistinctEndedWeeksAsync(LeagueWeek current, CancellationToken ct);

// Infra: distinct weeks via EF, then filter "< current" + sort IN MEMORY —
// EF can't translate < on a value-converted VO (only == / OrderBy by whole VO). Set is tiny.
var weeks = await db.LeagueStandings.Select(s => s.Week).Distinct().ToListAsync(ct);
return weeks.Where(w => w.Start < current.Start).OrderBy(w => w.Start).ToList();

Only weeks that have standings are candidates, so we never settle a truly empty week. Already-settled ones are no-ops via the marker — so this one query is all the new persistence we need (no schema change, no migration).
